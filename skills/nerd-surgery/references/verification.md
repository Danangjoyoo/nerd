# Verification Knowledge

Before stating success:

1. Identify the command or observation that proves the exact claim.
2. Run the complete proof fresh after the final change.
3. Read its exit status and full relevant output.
4. Map the evidence to the original symptom and acceptance criteria.
5. State only what the evidence supports and list any remaining gap.

A previous run, partial suite, code inspection, or expected outcome is not fresh proof.
