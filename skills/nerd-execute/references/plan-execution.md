# Plan Execution Knowledge

1. Load the complete approved plan and inspect it against repository reality.
2. Raise a material mismatch before implementation; otherwise map mechanical steps into outcome checkpoints.
3. Order checkpoints by dependency and proof. Keep tests with the behavior they verify.
4. Activate one checkpoint at a time.
5. Follow its required steps, run targeted proof, and advance only when it passes.
6. Stop for missing authority, ambiguous blockers, or repeated failed proof.
7. Run the milestone exit proof after every checkpoint is verified.
8. Run the full completion proof before presenting the branch result.

Do not skip plan requirements silently. Small mechanical regrouping is allowed when it preserves the approved outcome; architectural or deliverable changes require confirmation.
