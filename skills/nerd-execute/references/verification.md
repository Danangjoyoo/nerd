# Build Verification Knowledge

Before Build Result:

1. List every acceptance criterion from the Build Contract.
2. Identify the fresh command, artifact, or observation that proves each one.
3. Run the complete relevant commands after the final change.
4. Read exit status and output; do not infer success from partial checks.
5. Map evidence to each criterion.
6. Report unsupported or unavailable proof under Not verified.

Never call a checkpoint, milestone, or build complete while required acceptance lacks fresh evidence.
